---
title: 发布检查清单
description: 用于安全发布共享 Admin9 产品栈并确保开发者文档完整性的发布前检查项。
---

在打标签或部署发布前，可将本清单作为面向开发者的最后一道闸门。

## 1. 验证环境配置

- 确认 `APP_URL` 与真实公网域名一致
- 确认数据库、缓存、会话与队列驱动符合生产要求
- 若使用 Horizon 或队列任务，确认 Redis 可用
- 确认邮件提供商凭据有效
- 确认支付提供商凭据与 webhook 密钥已配置
- 确认社交登录凭据与回调 URL 正确
- 若部署包含默认路由集之外的身份提供方能力，需显式确认相关 issuer 与授权服务器配置

## 2. 验证运营功能

- 登录 `/admin`
- 登录 `/dashboard`
- 验证商品、套餐、货币与支付提供商种子数据或生产数据存在
- 验证法律页面可正常渲染
- 验证博客与路线图路由可访问
- 验证发票生成功能可用

## 3. 验证关键流程

- 注册
- 邮箱验证
- 社交登录
- 订阅结账
- 一次性商品结账
- 套餐变更流程
- 若启用，验证余额充值流程
- 验证已启用支付提供商的 Webhook 处理

## 4. 验证平台集成

- 队列 Worker 或 Horizon
- 邮件投递
- 若启用，验证 reCAPTCHA
- 若启用，验证 Twilio 验证
- 若启用，验证分析脚本
- 本地化路由
- 主题切换

## 5. 执行质量门禁

```bash
vendor/bin/phpunit
vendor/bin/phpstan analyse
vendor/bin/pint
npm run build
```

如果使用 Deployer 发布，执行前还应确认目标主机与部署配置：

```bash
php dep deploy
```

## 6. 验证文档完整性

发布前请确保外部文档站覆盖：

- 安装
- 环境配置
- 架构概览
- 支付
- 认证与 OAuth/OIDC
- 本地化
- 部署
- 破坏性变更的发布说明或升级说明

## 7. 标注已知不一致项

如果旧文档或内部笔记仍引用过时框架版本，请在发布面向公众的文档前先消除这些不一致。
